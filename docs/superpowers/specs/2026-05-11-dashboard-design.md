# Dashboard 메뉴 설계

**날짜:** 2026-05-11  
**상태:** 확정  
**스코프:** 3탭 구조 (투자 탭 스펙아웃)

---

## 목표

가계부 앱에 **DASHBOARD** 메뉴를 추가한다. 지출/수입/자산의 큰 흐름과 분석을 한 화면에서 파악할 수 있도록 한다.

- 연간 추세 중심, 기간 선택 가능
- 순자산과 투자 포트폴리오 요약이 핵심 지표
- 탭별로 영역을 분리해 깊은 분석 제공

---

## 구조

### 탭 구성

```
Dashboard
├── 개요 탭  (기본 진입 탭)
├── 지출 탭
└── 자산 탭
```

투자 탭은 현재 스코프에서 제외. 투자 요약은 개요 탭에 포함.

### 기간 선택기 (전역)

- 위치: 개요 탭 상단 우측
- 선택지: 이번달 / 이번 분기 / 올해 / 커스텀(날짜 범위 직접 입력)
- 동작: 선택 시 모든 탭의 차트·수치에 동일하게 적용
- 예외: 자산 탭의 자산 구성 도넛 및 항목 리스트는 항상 최신 기준

## 탭별 설계

### 1. 개요 탭

스크롤 순서 (위 → 아래):

| # | 컴포넌트 | 내용 |
|---|---------|------|
| ① | 기간 선택기 | 전역 기간 조정 |
| ② | 순자산 히어로 카드 | 총자산 - 부채 = 순자산, 전월/전년 대비 변화금액 + 방향 |
| ③ | 수지 요약 + 투자 요약 (2컬럼) | 수입/지출/저축액 \| 원금/평가금/수익률 |
| ④ | 순자산 변화 바 차트 | 기간 내 월별 순자산 추이 (12개월) |
| ⑤ | 지출 TOP 5 카테고리 | 수평 미니 바 차트 + 금액 |
| ⑥ | 최근 거래 3~5건 | 지출/수입/투자 최신 순, "더보기" 탭 시 해당 탭으로 이동 |

---

### 2. 지출 탭

스크롤 순서:

| # | 컴포넌트 | 내용 |
|---|---------|------|
| ① | 총 지출 헤더 카드 | 기간 총 지출, 전월 대비 증감, 기간 내 월평균 |
| ② | 카테고리별 도넛 차트 | 지출 카테고리 비중, 범례 탭 시 카테고리 상세 이동 |
| ③ | 월별 지출 바 차트 | 기간 내 월별 지출 추이, 이번달 강조 |
| ④ | 카테고리 상세 리스트 | 금액 + 비중 바 + 전월 대비 ▲▼, 탭 시 거래 목록 이동 |
| ⑤ | 최대 단건 지출 TOP 5 | 가장 큰 단건 지출 항목 |

---

### 3. 자산 탭

스크롤 순서:

| # | 컴포넌트 | 내용 |
|---|---------|------|
| ① | 순자산 헤더 | 총자산 / 부채 / 순자산 3단 표시, 전년 대비 성장액 |
| ② | 자산 구성 도넛 차트 | 현금/예금 · 투자자산 · 부동산 · 기타 비중 + 금액, 항상 최신 기준 |
| ③ | 순자산 장기 라인 차트 | 가입 이후 ~ 현재 순자산 변화, 기간 선택기 연동 |
| ④ | 자산 항목별 리스트 | 개별 자산 항목 + 금액, 카테고리 색상 점으로 구분 |
| ⑤ | 부채 현황 | 부채 항목별 + 총 부채, 부채 없으면 섹션 전체 숨김 |

## 필요한 백엔드 API

**신규 백엔드 API 없음.** 모든 데이터는 기존 API를 조합해 클라이언트에서 집계한다.

| 대시보드 데이터 | 기존 API | 집계 방법 |
|----------------|----------|-----------|
| 수입/지출/투자 합계 | `AccountService.getAccounts(divisionId, strtDt, endDt)` × 3 병렬 | 클라이언트 price 합산 |
| 카테고리별 지출 비중 | `CategoryService.getCategorySum(divisionId=3, strtDt, endDt)` | 그대로 사용 |
| 월별 지출 추이 | `AccountService.getAccounts(divisionId=3, strtDt, endDt)` | `accountDt.substring(0,6)` 기준 월별 그룹핑 |
| 최대 단건 지출 TOP5 | `AccountService.getAccounts(divisionId=3, strtDt, endDt)` | price 내림차순 정렬 후 take(5) |
| 현재 자산 구성 | `MyAssetService.getMyAssets(strtDt=today, endDt=today)` | `data` 맵의 `assetTotSumPrice` 사용 |
| 순자산 이력 | `MyAssetService.getMyAssetSum(strtDt, endDt)` | 날짜별로 총자산(`assetId != '0' && != '6'`) - 부채(`assetId == '6'`) |
| 최근 거래 | `AccountService.getAccounts(strtDt, endDt)` | `accountDt` 내림차순 정렬 후 take(5) |

### assetId 규칙 (MyAssetSumResponse)

| assetId | 의미 |
|---------|------|
| `'0'` | 더미 합계 행 — 무시 |
| `'6'` | 부채 — 순자산 계산 시 차감 |
| 그 외 | 실제 자산 항목 |

순자산 이력 계산:
```
net_worth(date) = sum(sumPrice where assetId ∉ {'0','6'}) - sum(sumPrice where assetId == '6')
```
