### 1-1. Toss (토스) — 한국 핀테크 미니멀리즘의 기준

**브랜드 포지셔닝**: "금융을 쉽게" — 복잡한 금융 정보를 극단적으로 단순화

**색상 팔레트**

| 역할 | 색상명 | HEX |
|------|--------|-----|
| 배경 메인 | Toss Dark | `#0D0D0F` |
| 배경 서브 (카드) | Elevated Surface | `#1A1A1C` |
| 배경 3단계 | Card Border Zone | `#2A2A2C` |
| 브랜드 강조색 | Toss Blue | `#3182F6` |
| 강조색 라이트 | Blue Light | `#5EA8FF` |
| 텍스트 주요 | White Primary | `#FFFFFF` |
| 텍스트 보조 | Gray 400 | `#9EA3AD` |
| 텍스트 비활성 | Gray 600 | `#5B5F6A` |
| 구분선 | Separator | `#2C2C2E` |

**카드/컴포넌트 스타일**

- **플랫 카드**: 그림자 없음, 배경색으로만 깊이감 표현 (`#1A1A1C` on `#0D0D0F`)
- **테두리 없음**: BorderRadius 12px, 테두리선 미사용
- **내부 여백**: 수직 20px, 수평 20px (일관성이 핵심)
- **구분선**: `#2C2C2E` 1px, Divider로 섹션 분리

**타이포그래피**

- **폰트**: Pretendard (한국 시장 사실상 표준, Variable 웨이트 지원)
- **금액 표시**: FontWeight.w700 이상, 금액은 숫자부와 단위("원")를 다른 크기로 분리
  - 메인 금액: 32~40px, Bold, White `#FFFFFF`
  - 단위 "원": 16~18px, Medium, Gray `#9EA3AD`
- **숫자 렌더링**: Tabular figures (`fontFeatures: [FontFeature.tabularFigures()]`)
- **카드 소제목**: 13~14px, Regular, Gray `#9EA3AD`

**레이아웃 구조**

- 홈 대시보드: 상단 총 자산 대형 숫자 → 스크롤로 카드 나열
- 카드 간 간격: 8px (밀도 높음, 빽빽하지 않음)
- 좌우 패딩: 20px 고정
- 섹션 헤더: 좌측 정렬, 17px Medium, `#9EA3AD`

**마이크로 인터랙션**

- 탭 시 카드 배경색 미세하게 변화 (ripple 없음, Opacity 방식)
- 숫자 변경 시 카운트업 애니메이션 (300ms ease-out)
- 페이지 전환: iOS style slide (FadeTransition 없이 SlideTransition)
- 로딩: Shimmer 스켈레톤 (Circular spinner 지양)

**토스의 핵심 설계 원칙**

1. 텍스트 위계로 시각적 계층 구성 (색상보다 크기·굵기 우선)
2. 강조색(`#3182F6`)은 CTA 버튼과 링크에만 제한적 사용
3. 아이콘: 선 굵기 균일한 Outlined 스타일, 20~24px
4. 여백이 디자인 요소 — 빈 공간을 두려워하지 않음

### 1-2. Robinhood — 투자 앱, 데이터 중심 다크 UI

**브랜드 포지셔닝**: "투자 민주화" → 2024년 리브랜딩으로 "성숙한 투자 플랫폼"으로 전환

**2024년 리브랜딩 핵심 변화**

- 기존 민트 그린 → "Robin Neon" (밝은 Yellow-Green) 으로 대체
- 블랙 + 화이트 + 뉴트럴 기반에 Robin Neon 포인트 방식으로 전환
- "성숙한 투자자"를 위한 소피스티케이티드 느낌 강화

**색상 팔레트**

| 역할 | 색상명 | HEX |
|------|--------|-----|
| 배경 메인 | True Black | `#000000` |
| 배경 서브 | Deep Gray | `#111111` |
| 배경 카드 | Card Dark | `#1C1C1C` |
| 브랜드 강조색 | Robin Neon | `#C8F135` (Yellow-Green) |
| 수익 표시 | Profit Green | `#00C805` |
| 손실 표시 | Loss Red | `#FF5000` (Orange-Red, 충격 완화) |
| 텍스트 주요 | White | `#FFFFFF` |
| 텍스트 보조 | Neutral 400 | `#8C8C8C` |
| 차트 그래디언트 (상승) | Green Grad | `#00C805` → `#004D03` |

**카드/컴포넌트 스타일**

- **순수 플랫**: True Black 위에 `#1C1C1C` 카드, 그림자 완전 없음
- **차트 영역**: 전체 화면 폭 확장, 하단 그래디언트 페이드 (수익=초록, 손실=빨강)
- **데이터 밀도**: 텍스트 14px까지 허용, 정보 밀도 높음
- **핵심 지표**: 포트폴리오 총액을 AppBar 바로 아래 대형(40px+) 숫자로 고정 표시

**타이포그래피**

- **폰트**: SF Pro / Roboto 유사 계열
- **금액**: 44~48px, FontWeight.w700, 수익 방향에 따라 색상 변경
- **차트 레이블**: 11px, Regular, Gray

**레이아웃 구조**

- 홈: 포트폴리오 숫자 → 가격 차트(풀폭) → 워치리스트 카드
- 거래 목록: 날짜 구분 없이 연속 리스트, 좌=종목명 우=수익률

**마이크로 인터랙션**

- 차트 터치: 날짜/가격 툴팁 슬라이드 (드래그 반응형)
- 매수/매도 완료: 전화면 컨페티(Confetti) 애니메이션
- 숫자 업데이트: Roll 애니메이션 (자릿수별 독립 슬라이드)
- Robin Neon 버튼: 탭 시 약간 scale-down (0.95)

**Robinhood의 핵심 설계 원칙**

1. 차트가 주인공 — 데이터 시각화 우선
2. 수익=자연스러운 초록, 손실=덜 공격적인 Orange-Red (심리적 완충)
3. 브랜드 컬러(Robin Neon)는 행동 유도(CTA)에만 집중 사용
4. 숫자 정확성과 빠른 인식이 최우선

### 1-3. Revolut — 유럽 핀테크, 사이버펑크 다크 미학

**브랜드 포지셔닝**: 글로벌 금융 슈퍼앱, 테크 포워드 프리미엄

**색상 팔레트**

| 역할 | 색상명 | HEX |
|------|--------|-----|
| 배경 메인 | Shark Dark | `#191C1F` |
| 배경 서브 | Surface Dark | `#22262B` |
| 배경 카드 | Elevated Card | `#2C3138` |
| 브랜드 강조색 | Revolut Blue | `#0075EB` |
| 보조 강조색 | Deep Purple | `#6E4CE5` |
| 그래디언트 시작 | Blue Grad | `#0B84F6` |
| 그래디언트 끝 | Purple Grad | `#6E4CE5` |
| 텍스트 주요 | White | `#FFFFFF` |
| 텍스트 보조 | Cornflower Gray | `#81B2F1` |
| 텍스트 비활성 | Muted | `#5A6070` |
| 수입 | Teal | `#00D09C` |
| 지출 | Coral | `#F08389` |

**카드/컴포넌트 스타일**

- **글래스모피즘 카드**: `backdrop-filter: blur(20px)`, 반투명 배경 (`rgba(44, 49, 56, 0.8)`)
- **그래디언트 경계선**: 카드 테두리에 `#0B84F6` → `#6E4CE5` 1px 그래디언트
- **뱅킹 카드**: 수평 그래디언트 (블루-퍼플-핑크), 실물 카드 3D 회전 애니메이션

**타이포그래피**

- **폰트**: Inter (서체 선명도 최우선, Revolut 공식 채택)
- **금액 표시**: 36~44px, FontWeight.w600, White
- **섹션 레이블**: 12px Uppercase Letter-spacing 0.8px, Gray
- **카드 내부**: 굵기 대비 활용 — 주요 정보 w600, 보조 정보 w400

**레이아웃 구조**

- 홈: 카드 이미지(실물 카드 3D) → 총 잔액 → 빠른 액션 버튼 → 거래 내역
- 그래프 영역: 네온 선 그래프, 배경 그래디언트 fill

**마이크로 인터랙션**

- 카드 스와이프: 3D 회전 + 금속 광택 효과
- 결제 완료: 녹색 체크 원형 → 파동 효과 (ripple outward)
- 환율 업데이트: 숫자 Flip 애니메이션
- 드래그로 금액 선택: 슬라이더 글로우 효과

**Revolut의 핵심 설계 원칙**

1. 프리미엄 시각 경험 — 글래스, 그래디언트, 3D로 일반 앱과 차별화
2. 블루-퍼플 그래디언트가 브랜드 시그니처
3. 사용자가 테마를 선택 가능한 개인화 강조
4. 다국어/다통화 → 레이아웃 유연성 최우선

## 2. 공통 디자인 패턴 요약

### 2-1. 배경색 계층 구조 (3단계 공통)

모든 앱이 3단계 배경 계층을 사용함:

```
Level 0: 앱 배경    #000000 ~ #191C1F  (가장 어두움)
Level 1: 섹션/카드  #111111 ~ #22262B  (미세하게 밝음)
Level 2: 인터랙티브 #1C1C1C ~ #2C3138  (카드 위 요소)
```

순수 `#000000` (True Black)은 Robinhood, Cash App처럼 "임팩트 강조"형에 사용.
`#0D0D0F` ~ `#191C1F` 같은 "Soft Dark"는 Toss, Revolut처럼 "장시간 사용" 앱에 적합.

### 2-2. 강조색 사용 원칙

| 앱 | 강조색 | 사용 범위 |
|----|--------|---------|
| Toss | `#3182F6` (Blue) | CTA 버튼, 링크, 진행바 |
| Robinhood | `#C8F135` (Neon Yellow-Green) | CTA만 — 극히 제한적 |
| Revolut | `#0B84F6`→`#6E4CE5` (Blue-Purple Grad) | 카드 경계, 그래프, 버튼 |
| Cash App | `#00D54B` (Malachite Green) | 모든 CTA, 성공 상태 |

공통 원칙: **강조색은 1~2가지로 제한, 의미 있는 곳에만 사용**.

### 2-3. 숫자/금액 강조 방법

- **크기 위계**: 메인 금액 40~56px (일반 텍스트의 3~4배)
- **굵기**: FontWeight.w700 또는 w800 (Bold 이상)
- **색상**: 순수 White `#FFFFFF` — 배경과 최대 대비
- **단위 분리**: 금액 숫자와 "원"/"$" 등 단위를 다른 크기로 렌더링
- **Tabular Figures**: 자릿수 변경 시 레이아웃 밀림 방지 (필수)
- **음수/손실**: 빨간 계열, 양수/수익: 초록 계열

### 2-4. 카드 스타일 3가지 유형

| 유형 | 특징 | 채택 앱 |
|------|------|---------|
| 플랫 다크 | 그림자 없음, 배경 차이만으로 깊이 표현 | Toss, Cash App |
| 글래스모피즘 | 반투명 + 블러 + 그래디언트 테두리 | Revolut (고급형) |
| 순수 블랙 플랫 | 구분선 없이 색상 없음, 극단적 미니멀 | Robinhood |

### 2-5. 아이콘 스타일

- **Outlined (선형)**: Toss, Revolut — 세련, 가벼움
- **Filled (채움)**: Cash App — 친근, 강하고 빠른 인식
- **크기**: 20~24px 표준, 홈 빠른 액션 버튼은 28~32px
- **색상**: 배경보다 밝은 Gray 또는 강조색 단색

### 2-6. 진행률 바 (Progress Bar)

- **높이**: 4~6px (얇음) — 두꺼운 바는 구식
- **배경**: `rgba(255,255,255,0.1)` (반투명 흰색)
- **채움색**: 의미 색상 또는 강조색 그래디언트
- **끝처리**: BorderRadius 100px (완전 원형)

### 2-7. 타이포그래피 공통 원칙

| 레벨 | 사용처 | 크기 | 굵기 |
|------|--------|------|------|
| Display | 메인 금액, 잔액 | 40~56px | w700+ |
| Heading | 카드 제목, 섹션명 | 18~24px | w600 |
| Body | 거래 내역, 설명 | 14~16px | w400 |
| Caption | 날짜, 부제목, 비율 | 11~13px | w400 |

## 3. 커플 가계부 적용 시 고려사항

### 3-1. 핀테크 전문성과 커플 감성의 균형

**조화 방법**:

- 배경과 기본 UI는 핀테크 다크 스타일 유지 (신뢰감, 집중력)
- 강조색에 따뜻한 계열을 섞음: 순수 파란색 대신 Teal(청록), 또는 Warm Coral 포인트
- 사용자 아바타/프로필 이미지 원형 표시 — 인물감으로 친근함 부여
- 커플 공유 데이터 시각화 시 사용자별 색상 대비 (파랑 vs 코랄/핑크)

### 3-2. 공유/협업 UI 패턴

- **사용자 구분 배지**: 아바타 + 이름 태그를 카드 좌측에 고정
- **색상으로 사용자 구분**: 사용자 1은 Teal, 사용자 2는 Coral/Pink (일관성)
- **비율 시각화**: 두 사용자의 지출 비율을 Dual Progress Bar로 표현
- **공유 감각**: 총합과 개별 분리 표시

## 5. 추천 방향

### 최종 선택: Palette E — "Teal Fusion"

#### 선정 근거

**1. 이 앱의 고유 요구사항에 가장 부합**

커플 가계부는 냉철한 투자자 앱이 아님. "함께 재정을 관리하는 성인 커플"이 타겟이며, 이 팔레트는 핀테크 전문성(다크 배경, 명확한 숫자)과 따뜻한 감성(Teal, Indigo 톤)을 동시에 제공.

**2. 현존 레퍼런스와 차별화**

- Toss 스타일 배경 구조(`#0D1117` 계열) 차용 → 장시간 사용 적합
- Revolut 그래디언트 접근법 차용 → 시각적 개성
- 강조색(Teal `#2DD4BF`)은 레퍼런스 앱 어디에도 없는 독자 포지션

**3. 의미론적 색상 재해석**

기존 앱의 색상 시스템(수입=파랑, 지출=빨강, 투자=주황)을 유지하되 더 세련된 채도의 Soft 버전으로 업그레이드. 현재 코드의 색상 로직과 자연스럽게 연계됨.

**4. Flutter 구현 현실성**

- 글래스모피즘 없이도 충분한 깊이감 (Palette B 대비 성능 우수)
- True Black 아님 → OLED 번인 위험 없이 편안한 배경 (`#0D1117`)
- Material 3 ThemeData에 자연스럽게 매핑 가능
- Pretendard (또는 Noto Sans KR) 조합으로 한국어 최적화

**5. 사용자 구분 컬러 시스템 완성도**

강원(Teal `#2DD4BF`) + 정윤(Pink `#F472B6`) 조합은 시각적으로 명확히 구분되면서 커플 앱에 어울리는 따뜻한 대비를 제공.

#### 구현 시 핵심 지침

1. **배경 계층 3단계 엄수**: `#0D1117` / `#161B22` / `#21262D`
2. **금액 숫자 최우선**: Display 텍스트 40px+, w700, `#E6EDF3`
3. **Teal 그래디언트 제한 사용**: 홈 대시보드 수입 카드, CTA 버튼에만
4. **Progress Bar 통일**: 높이 4px, BorderRadius 100, 배경 `rgba(255,255,255,0.08)`
5. **Pretendard 폰트 적용**: fontFeatures Tabular Figures 필수
6. **아이콘**: Material Symbols Outlined, 크기 24px 표준화
